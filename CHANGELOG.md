# Changelog

All notable changes are documented in this file.

The project follows semantic versioning after the initial `0.x` development series.

## 0.2.0

- Added pre-device execution-contract gates for rules, exact APIs, mutation assets, requested IDs, and page anchors.
- Unified request matching on exact host, path, and HTTP method.
- Added fresh device Wi-Fi prefix discovery, common-subnet selection, and source-bound phone route probes.
- Added pre-mutation durable proxy snapshots, SSID-safe restoration, owned-process stopping, and cross-process workspace/store locks.
- Defaulted report hosting to loopback and redacted URL query values plus raw before/after evidence.
- Separated non-final visual prechecks from mandatory semantic or human review.
- Added an opt-in physical Android/iOS/HarmonyOS regression matrix.

## 0.1.0

- Initial public release.
- Android, iOS, and HarmonyOS device-control adapters.
- Single-call multi-lane case execution with readiness gates and stage budgets.
- Managed mitmproxy lifecycle with verified device proxy configuration and explicit manual-close reminders.
- Append-only, redacted modified-response evidence.
- HTML, Markdown, JSON, JSONL, screenshot, trace, and integrity-manifest reporting.
- Navigation knowledge, visual comparison, WDA recovery, diagnostics, and report serving.
